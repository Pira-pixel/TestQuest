# Веб-бенчмарк инструмент

Программа для нагрузочного тестирования веб-серверов с поддержкой асинхронных запросов.

## Возможности

- Тестирование нескольких хостов одновременно
- Асинхронные запросы для максимальной производительности
- Поддержка различных форматов вывода (текст, JSON, CSV)
- Валидация входных параметров
- Сохранение результатов в файл
- Подробная статистика по каждому хосту

## Установка

1. Клонируйте репозиторий:
bash
git clone <ваш-репозиторий>
cd benchmark-tool

в терминале пропишем:
pip install aiohttp>=3.8.0
pip install aiofiles>=23.0.0
pip install request

# Базовое использование:
# Использование
Запуск из папки TestQuest:
# Перейти в папку проекта
cd TestQuest

# Тестирование одного хоста
python bench.py -H https://ya.ru

# Тестирование нескольких хостов
python bench.py -H https://ya.ru,https://google.com,https://github.com -C 3

# Чтение хостов из файла
python bench.py -F hosts.txt

# Сохранение результатов
python bench.py -H https://ya.ru,https://google.com -O results.json --format json

# Все параметры:
python bench.py --help

# Параметры командной строки
Параметр	    Описание	    Пример
-H,             --hosts	        Список хостов через запятую	-H https://ya.ru,https://google.com
-F,             --file	        Файл со списком хостов	-F hosts.txt
-C,             --count	        Количество запросов на хост	-C 10
-O,             --output	    Файл для сохранения результатов	-O results.txt
--format	    Формат вывода (text/json/csv)	--format json

# Формат вывода
Текстовый формат:
============================================================
Host: https://ya.ru
------------------------------------------------------------
Success:        10
Failed:         0
Errors:         0
Min time:       0.0456 сек
Max time:       0.1234 сек
Avg time:       0.0789 сек
============================================================

# JSON формат:
{
  "https://ya.ru": {
    "host": "https://ya.ru",
    "success": 10,
    "failed": 0,
    "errors": 0,
    "min": 0.0456,
    "max": 0.1234,
    "avg": 0.0789
  }
}

# CSV формат:
csv
Host,Success,Failed,Errors,Min,Max,Avg
https://ya.ru,10,0,0,0.0456,0.1234,0.0789

# Метрики
Success: Количество успешных запросов (статус 2xx/3xx)

Failed: Количество запросов с ошибкой сервера (статус 4xx/5xx)

Errors: Количество запросов с сетевыми ошибками

Min: Минимальное время ответа

Max: Максимальное время ответа

Avg: Среднее время ответа

## Особенности реализации:

1. **Асинхронность**: Используется `asyncio` и `aiohttp` для конкурентного выполнения запросов
2. **Валидация**: Проверка формата URL с использованием регулярных выражений
3. **Обработка ошибок**: Все ошибки обрабатываются с понятными сообщениями
4. **Гибкий вывод**: Поддержка текстового, JSON и CSV форматов
5. **Аргументы командной строки**: Используется `argparse` для обработки параметров
6. **Чтение из файла**: Поддержка чтения списка хостов из файла
